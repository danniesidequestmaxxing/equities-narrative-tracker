Why it looks like this. Every dynamic value is MarkdownV2-escaped at the boundary, tickers render in monospace so $BRK.B can't break parsing, and a parse error silently falls back to plain text (never a dropped call). Calls + retractions are priority-0 and bypass alert-coalescing. The ops channel is separate so health/budget noise never lands in the group. A web dashboard (read-only: outcome history, credibility leaderboard, cost) is the planned next surface — Next.js on Vercel reading the same Postgres — but it isn't built yet.
